# 알고리즘 - 큐

## 1. 큐(Queue)

**특성**

선입선출구조 (FIFO: First In First Out)

**구조**

머리: Front (삭제)

꼬리: Rear (삽입)

**기본 연산**

삽입: enQueue

삭제: deQueue

**주요 메소드**

- offer()
- poll()
- peek()
- isEmpty()
- size()

**큐 활용**

버퍼: 순서대로 입출력/전달되어야 하므로 FIFO 방식의 큐 활용

## 2. 큐 활용 - 마이쮸

**마이쮸 나눠주기 시뮬레이션**

![image.png](image.png)

```java
import java.util.ArrayDeque;
import java.util.Queue;

public class QueueMyChewTest {

	public static void main(String[] args) {

		int N = 20; // 현재 마이쮸 개수
		int person = 1; // 대기열에 들어갈 다음 사람의 번호
		
		Queue<int[]> queue = new ArrayDeque<>();
		queue.offer(new int[] {person, 1});
		
		while (N > 0) {
			int[] p = queue.poll();
			int availCnt = (N >= p[1]) ? p[1] : N;
			N -= availCnt;
			
			if (N == 0) System.out.println(p[0] + "번 사람이 마지막 마이쮸를 가져갑니다. 가져간 개수 : " + availCnt);
			else {
				System.out.println(p[0] + "번 사람이 마지막 마이쮸를 가져갑니다. 가져간 개수 : " + availCnt);
				++p[1]; // 다음에는 마이쮸 +1개만큼 더 받기
				queue.offer(p);
				queue.offer(new int[] {++person, 1});
			}
		}
	}
}

/*
1번 사람이 마지막 마이쮸를 가져갑니다. 가져간 개수 : 1
1번 사람이 마지막 마이쮸를 가져갑니다. 가져간 개수 : 2
2번 사람이 마지막 마이쮸를 가져갑니다. 가져간 개수 : 1
1번 사람이 마지막 마이쮸를 가져갑니다. 가져간 개수 : 3
3번 사람이 마지막 마이쮸를 가져갑니다. 가져간 개수 : 1
2번 사람이 마지막 마이쮸를 가져갑니다. 가져간 개수 : 2
4번 사람이 마지막 마이쮸를 가져갑니다. 가져간 개수 : 1
1번 사람이 마지막 마이쮸를 가져갑니다. 가져간 개수 : 4
```

## 3. 재귀 - 피보나치 수열

F(0) = 0, F(1) = 1

F(i) = F(i - 1) + F(i - 2) for i ≥ 2

재귀함수 구현

```java
fibo(n)
	if n < 2 : RETURN n
	else     : RETURN fibo(n - 1) + fibo(n - 2)
```

**⇒ 엄청난 중복 호출이 존재!!** 

```java
// nCr의 경우의 수 구하기
// n! / r!(n -r)! 대신 재귀로 설계

// nCr의 경우의 수 구하기

public class RecurCombinationTest {
	public static int combination(int n, int r) {
		if (n < r) throw new IllegalArgumentException();
		if (r == 0 || n == r) return 1;
		return combination(n - 1, r - 1) + combination(n - 1, r);
	}

	public static void main(String[] args) {
		System.out.println(combination(4, 1));
		System.out.println(combination(4, 2));
		System.out.println(combination(4, 3));
		System.out.println(combination(4, 4));
	}
}

/*
4
6
4
1
*/
```
